## 活动方案分析

适用活动：保时捷联动。

报告模块标题：`保时捷联动方案分析`。

分析规则：

- 基础候选按 `1 万方斯等效环石 < 8` 的档位，从等效值低到高逐步改用环石；`1 万方斯等效环石 >= 8` 的档位默认保留方斯支付。
- 若方斯底线仍不足，或用户指定强保方斯、环石充足，补列 9、10 档或全环石等扩展补差方案。
- 若低成本方斯礼包能解决缺口，优先列礼包补差，再考虑双倍首充。
- 输出 3-6 条与约束最相关的方案；若用户指定环石上限或方斯底线，围绕约束列相邻方案。

| 方案 | 联动环石 | 联动方斯 | 适合情况 |
| --- | ---: | ---: | --- |
| 全方斯 | 0 | 19,880,000 | 环石极缺、方斯极富裕 |
| 15 | 2,200 | 15,680,000 | 优先使用最划算档位保方斯 |
| 14-15 | 4,200 | 12,180,000 | 方斯压力中等 |
| 13-15 | 6,000 | 9,180,000 | 需要明显保方斯 |
| 12-15 | 7,600 | 6,680,000 | 方斯压力较大 |
| 11-15 | 9,000 | 4,680,000 | 强保方斯 |
| 8, 11-15 | 9,600 | 3,880,000 | 接近 `<8` 档位全环石 |
