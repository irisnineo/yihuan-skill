## 活动方案分析

适用活动：保时捷联动盲盒。

报告模块标题：`保时捷联动盲盒方案分析`。

分析规则：

- 基础候选按 `每环石方斯 > 1,250` 的抽数，从等效值高到低逐步改用环石；`每环石方斯 <= 1,250` 的抽数默认保留方斯支付。
- 若方斯底线仍不足，或用户指定强保方斯、环石充足，补列 9、10 抽或全环石等扩展补差方案。
- 输出 3-6 条与约束最相关的方案；若用户指定环石上限或方斯底线，围绕约束列相邻方案。

| 方案 | 联动环石 | 联动方斯 | 适合情况 |
| --- | ---: | ---: | --- |
| 全方斯 | 0 | 19,880,000 | 环石极缺、方斯极富裕 |
| 15 | 2,200 | 15,680,000 | 优先使用最划算档位保方斯 |
| 14-15 | 4,200 | 12,180,000 | 方斯压力中等 |
| 13-15 | 6,000 | 9,180,000 | 需要明显保方斯 |
| 12-15 | 7,600 | 6,680,000 | 方斯压力较大 |
| 11-15 | 9,000 | 4,680,000 | 强保方斯 |
| 8, 11-15 | 9,600 | 3,880,000 | 接近高等效抽数全环石 |

报告表结构：

| 列 | 内容 |
| --- | --- |
| 方案 | 抽数组合和支付方式，推荐方案在方案文本后加（推荐），例如 13-15 抽用环石（推荐） |
| 联动环石 | 该方案环石消耗 |
| 联动方斯 | 该方案方斯消耗 |
| 版本末环石 | 扣除所有目标后的版本末环石 |
| 版本末方斯 | 扣除所有目标后的版本末方斯；不足时写缺口 |
| 说明 | 一句短评 |
