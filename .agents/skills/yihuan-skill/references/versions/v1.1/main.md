# 异环 1.1 版本主资料

用于记录 1.1 版本的特殊规则、活动档位、常见目标和计算公式，不设置固定玩家偏好或固定当前库存。

版本日期、资源收入、日期拆分、礼包上架和汇总校验见 `progress.md`；分析时优先读取 `progress.md` 的资源数据。

支付原则：

- `1 万方斯等效环石` 越低，越优先用环石替换方斯。
- 需要保方斯安全垫时，从高额档或方斯效率最差的档位开始替换。
- 若低成本方斯礼包能解决缺口，先列礼包补差，再考虑双倍首充。

### 连续后段方案

默认只分析连续后段：从某一档开始，到 15 档全部用环石支付；该档之前全部用方斯支付。用户明确要求自由组合时，才分析非连续组合。

```text
联动环石 = 起始档位至 15 档的环石总和
联动方斯 = 1 档至起始档位前一档的方斯总和
```

| 环石支付档位 | 联动环石 | 联动方斯 | 适合情况 |
| --- | ---: | ---: | --- |
| 无，全方斯 | 0 | 19,880,000 | 环石极缺、方斯极富裕 |
| 15 | 2,200 | 15,680,000 | 只替换最高额方斯档 |
| 14-15 | 4,200 | 12,180,000 | 小幅保方斯 |
| 13-15 | 6,000 | 9,180,000 | 均衡起点 |
| 12-15 | 7,600 | 6,680,000 | 方斯压力较大 |
| 11-15 | 9,000 | 4,680,000 | 明显保方斯 |
| 8-15 | 11,600 | 1,380,000 | 接近全环石 |

输出时选 3-6 条最相关方案；若用户指定环石上限或方斯底线，围绕约束列相邻方案。

## 1.1 常见开销与目标模板

常见都市/日常开销，只有在用户目标包含时才计入：

| 项目 | 消耗 |
| --- | ---: |
| 买房 | 7,500,000 |
| 猎人交易所 | 2,537,000 |
| 好感道具日常 | 51,000/天 x 35 天 = 1,785,000 |
| 全部计入合计 | 11,822,000 |

常见目标模板，按用户输入选择：

| 目标 | 规划 |
| --- | --- |
| 角色 X 个 | 按用户指定数量计算；1.1 角色 2 个按 2 个 S 角色计算 |
| 武器 X 把 | 按用户指定把数计算，每把约 60 抽 |
| 猎人交易所 | 常规购买 500 环石 + 40 三重钥匙 + 5 永恒的螺音，成本 2,537,000 方斯；用户明确不买时排除 |
| 保时捷联动 | 可选择参加满或指定档位 |
| 好感度 | 按用户输入或档案策略计入；备注写公式格式，如“按 15000 x 2 + 3000 x 7 = 51000/天，35 天合计。” |
| 买房 | 按用户输入计入；1.1 买房按 7,500,000 方斯计算 |
| 礼包 | 根据用户输入或玩家档案判断是否列入 |
| 双倍首充 | 根据用户输入或玩家档案判断是否列入 |

抽卡期望：

```text
2 个 S 角色：约 91.5 抽
1 把武器：约 60 抽
```

补差公式：

```text
角色补抽所需环石 = max(0, 角色需求抽数 - 可用限定骰子) x 160
武器补差所需环石 = max(0, 武器目标把数 x 60 - 可用三重钥匙) x 160
```
