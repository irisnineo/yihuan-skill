# 异环 1.1 版本主资料

本文件属于版本事实层，用于记录 1.1 版本的特殊规则和活动档位，不设置固定玩家偏好或固定当前库存。

版本日期、资源收入、日期拆分、礼包上架和收益汇总见 `progress.md`；分析时优先读取 `progress.md` 的资源数据。

支付原则：

- 档位 1-15 必须按顺序购买；每个已购买档位可单独选择环石或方斯支付。
- 默认只把 `1 万方斯等效环石 < 8` 的档位纳入环石替换分析，并按等效值从低到高逐步加入方案。
- `1 万方斯等效环石 >= 8` 的档位默认保留方斯支付；只有用户指定强保方斯、环石充足或方斯底线仍不足时，才列扩展补差方案。
- 若低成本方斯礼包能解决缺口，先列礼包补差，再考虑双倍首充。

### 默认方案分析

默认方案顺序：

```text
候选档位 = 1 万方斯等效环石 < 8 的档位
组合顺序 = 按 1 万方斯等效环石从低到高排列：15 -> 14 -> 13 -> 12 -> 11 -> 8
联动环石 = 环石支付档位的环石合计
联动方斯 = 其余已购买档位的方斯合计
```

| 方案 | 联动环石 | 联动方斯 | 适合情况 |
| --- | ---: | ---: | --- |
| 全方斯 | 0 | 19,880,000 | 环石极缺、方斯极富裕 |
| 15 | 2,200 | 15,680,000 | 优先使用最划算档位保方斯 |
| 14-15 | 4,200 | 12,180,000 | 方斯压力中等 |
| 13-15 | 6,000 | 9,180,000 | 需要明显保方斯 |
| 12-15 | 7,600 | 6,680,000 | 方斯压力较大 |
| 11-15 | 9,000 | 4,680,000 | 强保方斯 |
| 8, 11-15 | 9,600 | 3,880,000 | 接近 `<8` 档位全环石 |

输出时选 3-6 条与约束最相关的方案；若用户指定环石上限或方斯底线，围绕约束列相邻方案。若 `<8` 档位全改环石后仍不满足方斯底线，再补列 `1 万方斯等效环石 = 8` 的 9、10 档或全环石方案，并标记为扩展补差。报告模块标题写作 `保时捷联动方案分析`。
