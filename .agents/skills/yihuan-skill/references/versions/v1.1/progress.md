## 计算规则

资源口径默认使用 `进行中拆分`：查询日期默认使用当前日期，用户明确指定日期时才覆盖。
用户明确要求 `整版本预估`、`版本预估`、`还没开始玩` 或 `从零开始规划` 时，使用 `整版本预估`：不按查询日期拆分，直接使用汇总校验中的 1.1 整个版本资源。

```text
进行中拆分：
截至查询日已开放资源 = 可获得日期 <= 查询日期的资源 + 已累计每日资源
查询日后未开放资源 = 查询日期之后至版本结束仍可获得的资源
版本理论总资源 = 截至查询日已开放资源 + 查询日后未开放资源

整版本预估：
版本可用资源 = 当前库存 + 汇总校验中的整个版本资源 + 已确认购买的礼包资源
```

- 每日资源按包含首尾日期的自然日计算，并限制在 2026-05-28 至 2026-07-01 之间。
- 周期资源、一次性资源、活动资源按 `可获得日期` 判断是否已经开放。
- 活动奖励默认按活动第一天全部可获得计算。
- 礼包资源只表示“可购买资源”，不默认计入已获得资源；只有用户确认购买或预算规则明确时才计入。
- `待补日期` 项不计入已开放/未开放拆分；整版本汇总仍保留其数量用于校验。
