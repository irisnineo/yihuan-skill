## 活动方案分析

适用活动：Draco 盲盒。

报告模块标题：`Draco 盲盒方案分析`。

分析规则：

- 基础候选按 `每环石方斯 > 1,000` 的抽数，从等效值高到低逐步改用环石；`每环石方斯 <= 1,000` 的抽数默认保留方斯支付。
- 若方斯底线仍不足，或用户指定强保方斯、环石充足，补列 1-5 抽或全环石等扩展补差方案。
- 输出 3-6 条与约束最相关的方案；若用户指定环石上限或方斯底线，围绕约束列相邻方案。

| 方案 | 联动环石 | 联动方斯 | 适合情况 |
| --- | ---: | ---: | --- |
| 全方斯 | 0 | 11,800,000 | 环石极缺、方斯极富裕 |
| 11, 14-15 | 2,400 | 6,500,000 | 每环石方斯大于等于 2,200 的抽数用环石 |
| 10-15 | 4,250 | 2,600,000 | 每环石方斯大于 2,000 的抽数用环石 |
| 6-15 | 5,480 | 500,000 | 每环石方斯大于 1,000 的抽数用环石 |
| 全环石 | 5,980 | 0 | 强保方斯且环石充足 |

报告表结构：

| 列 | 内容 |
| --- | --- |
| 方案 | 抽数组合和支付方式，推荐方案在方案文本后加（推荐），例如 11-15 抽用环石（推荐） |
| 联动环石 | 该方案环石消耗 |
| 联动方斯 | 该方案方斯消耗 |
| 版本末环石 | 扣除所有目标后的版本末环石 |
| 版本末方斯 | 扣除所有目标后的版本末方斯；不足时写缺口 |
| 说明 | 一句短评 |
