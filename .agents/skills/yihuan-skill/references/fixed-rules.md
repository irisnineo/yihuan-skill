# 异环固定氪金体系与规则

本文件属于游戏概念层，用于记录异环资源规划的长期稳定规则。版本专属活动档位应放在对应版本参考文件中。

## 输入与档案

分析时按以下优先级确定玩家参数：

1. 用户本次输入。
2. 用户指定的 `profiles/<档案名>.md`。
3. 未指定档案时的默认档案 `profiles/nineo.md`。

当前库存必须来自用户输入或外部档案，不从版本文件读取。

### 档案结构

只有用户明确发出 `给{name}建档` 指令时，才建立新玩家档案。建档时复制 `profiles/profile-template.md`，保存为 `profiles/<name>.md`。

所有 `profiles/<档案名>.md` 至少应包含：

- 用户倾向
- 目标优先级
- 活动与商店
- 好感度

## 基础换算

```text
1 体力 = 1 都市活力 = 1000 方斯 = 1 环石
1 限定骰子 = 160 环石
1 三重钥匙 = 160 环石
```

- 除非用户明确说明可用，否则 `常驻骰子` 和其他常驻资源不计入限定角色或武器规划。
- 商店兑换、材料兑换、环石兑换体力/活力的详细规则见 `references/shop-exchange.md`。

## 抽卡道具规则

| 道具 | 可用卡池 | 使用规则 |
| --- | --- | --- |
| 版本礼包限定骰子 | 当期限定棋盘 | 限当期 |
| 限定骰子 | 限定棋盘 | 长期 |
| 三重钥匙 | 弧盘研摹 | 长期 |
| 常驻骰子 | 常驻池 | 长期 |
| 环石 | 限定棋盘、弧盘研摹 | 长期 |

抽卡返还资源已包含在期望抽数中，规划时不单独计入。

## 资源用途

| 资源 | 主要用途 | 规划原则 |
| --- | --- | --- |
| 环石 | 角色补抽、武器补差、活动支付、特殊兜底 | 优先保证限定角色 |
| 限定骰子 | 限定角色抽卡 | 先用限定骰子，再用环石补差 |
| 三重钥匙 | 武器抽卡 | 先用三重钥匙；环石优先保留给角色、活动和兜底 |
| 方斯 | 都市玩法、买房、载具、好感、猎人交易所、活动支付 | 先锁定必需开销，再考虑都市大件 |

## 日期拆分与版本收入规则

`references/versions/v<版本>/progress.md` 只维护具体版本的日期资源、礼包上架、汇总校验和版本特例。所有版本通用的日期拆分、礼包计入和整版本预估规则以本节为准。

资源计算默认使用 `进行中拆分`：查询日期默认使用当前日期，用户明确指定日期时才覆盖。用户明确要求 `整版本预估`、`版本预估`、`版本前`、`还没开始玩` 或 `从零开始规划` 时，使用 `整版本预估`，不按查询日期拆分。

用户写“版本 X + 当前资源”时，默认理解为规划版本 X 的期初库存；只有明确写“当前剩余资源”“已领取后的余量”或“版本进行中余量”时，才不重复叠加已开放资源。

```text
进行中拆分：
截至查询日已开放资源 = 可获得日期 <= 查询日期的资源 + 已累计每日资源
查询日后未开放资源 = 查询日期之后至版本结束仍可获得的资源
可用资源 = 当前库存 + 截至查询日已开放资源 + 查询日后未开放资源 + 计入礼包资源
版本理论总资源 = 截至查询日已开放资源 + 查询日后未开放资源

整版本预估：
可用资源 = 当前库存 + 汇总校验中的整版本资源 + 计入礼包资源
```

- 每日资源以具体版本 `progress.md` 的每日资源起止日期为准，按包含首尾日期的自然日计算。
- 周期资源、一次性资源和活动资源按 `可获得日期` 判断是否已经开放。
- 活动奖励默认按活动第一天全部可获得计算；若具体版本有分阶段领取规则，在该版本 `progress.md` 的版本特例中说明。
- 礼包资源只表示“可购买资源”，不默认计入已获得资源；只有用户确认购买或预算规则明确时才计入。
- 缺少可获得日期的资源标记为 `待补日期`，不计入已开放/未开放拆分；如整版本汇总保留其数量用于校验，应在该版本校验状态中说明。
- 缺少数量的数据标记为 `待补数据`，不计入资源统计或消耗统计。

## 卡池时间检查

如果版本 `progress.md` 存在 `卡池信息`，角色和武器目标必须同时做卡池结束时间检查。版本末资源够用，不代表较早结束的卡池也够用。

- `限定棋盘` 对应限定角色目标，消耗 `限定骰子` 和环石补差。
- `弧盘研摹` 对应武器目标，消耗 `三重钥匙` 和环石补差。
- `卡池信息` 的 `卡池名称` 应填写图中该卡池横幅标题，`卡池内容` 应填写图中明确展示的角色名或武器名；图中给出具体武器名时，使用具体武器名。
- 目标写“角色全拿”或“角色 X 个”且未指定具体角色时，按 `限定棋盘` 的时间顺序匹配版本内限定角色，角色名取 `卡池内容`。
- 武器目标未指定对应角色或武器时，优先按用户输入和档案偏好匹配；仍无法判断时，在报告中写明待确认。
- 检查某个卡池目标时，只计入该卡池结束时间前可获得的资源和计入的相关礼包；卡池结束后的资源归入后续目标或版本末余量。
- 多个卡池目标按结束时间顺序滚动计算，已经分配给前序目标的抽卡道具和环石不再重复计入后序目标。
- 如果卡池时间缺失，标记为 `待补日期`，报告中只做整版本估算，并说明未完成卡池结束时间检查。

## 抽卡成本规则

### 卡池机制

| 卡池类型 | 对应目标 | 抽卡道具 | 环石补差 | 保底 | 期望 |
| --- | --- | --- | ---: | ---: | ---: |
| 限定棋盘 | 限定角色 | 限定骰子 | 160 环石/抽 | 90 抽 | 45 抽 |
| 弧盘研摹 | 武器 | 三重钥匙 | 160 环石/抽 | 80 抽 | 60 抽 |

### 补差公式

```text
角色补抽所需环石 = max(0, 角色目标数量 x 45 - 可用限定骰子) x 160
武器补差所需环石 = max(0, 武器目标把数 x 60 - 可用三重钥匙) x 160
```

### 礼包

| 礼包类型 | 版本显示名 | 价格 | 内容 | 限购/刷新 | 备注 |

### 双倍首充

```text

双倍首充合计：1308 元 x 20 = 26160 环石
刷新周期：周年刷新
普通比例：648 元 = 8080 环石
```
