# 项目说明

## 文档语言

- 本项目的说明文档、技能文档、参考资料、玩家档案和报告模板默认使用中文撰写。
- 代码标识符、文件路径、命令、第三方专有名词和必要的技术关键词可以保留英文。

## 文档表达

- 文档默认写成可执行的规则、事实表和简短说明，优先服务资源计算和后续维护。
- 能用表格、清单或公式表达的信息，优先使用结构化形式；说明文字只保留判断依据、适用条件和会影响计算的例外，不重复解释表格已表达的内容。
- 规则文档优先使用正向约束、固定格式和自检项表达；反向禁用项只用于会破坏计算或格式的关键例外，避免在同一段落重复列举反向表述。
- 版本号写法按用途区分：文件夹目录、文件路径和表格列名使用 `v<版本>`，例如 `v1.1`；正文叙述、标题和报告文案使用 `<版本> 版本`，例如 `1.1 版本`。
- 同一规则只在一个位置维护。

## 原始资料边界

- 将 `sources/**` 视为原始资料和备份，只读处理。不修改、重命名、移动 `sources/**` 下的文件。
- 文档维护默认只修改 `.agents/skills/yihuan-skill/**` 下的文档。
- 资源规划和报告生成应以 `.agents/skills/yihuan-skill/references/**`、`.agents/skills/yihuan-skill/profiles/**` 和用户本次输入为正式依据。

## Git 操作

- 执行 `git push` 前，先核对本次 `commit` 内容和工作区状态，确认提交无误后再推送。
